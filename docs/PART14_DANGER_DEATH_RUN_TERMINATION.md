# Part 14 — Danger, Death and Run Termination

Part 14 adds bounded causal hazards, warnings, persistent injuries, treatment, recovery time, accumulated vulnerability, player death, and terminal run state.

Hazards are authored catalogue entries and cannot be invented by the LLM or Directors. Eligibility depends on shared simulation state such as location, supernatural pressure, anomaly exposure and prior vulnerability. The first eligible exposure surfaces a warning; repeated risky exposure can cause injury. Fatal outcomes require a severe hazard plus accumulated vulnerability and absence of the relevant learned warning, preventing arbitrary random death.

Death ends the current run and blocks ordinary actions. The terminal state exposes only the transition into another run. Part 15 will own cross-run memory and recurrence semantics; Part 14 only provides the clean terminal handoff.
