# Part 9 — Expanded NPC Autonomy and Purpose-Driven Behaviour

Part 9 adds a bounded purpose-ranking layer between authored NPC action libraries and the existing NPC Director.

The layer does not invent actions, destinations, canon, or transitions. It ranks already-authored and topology-valid candidates using persistent runtime needs, current obligations, place preferences and avoidance, weather preference, opening hours, world suitability metadata, and active-event context. The Director then receives a smaller purpose-aware set of choices plus an inspectable purpose plan explaining dominant need and scoring reasons.

This preserves the frozen movement guards while making Parts 1–8 causally useful to NPC decisions. Runtime needs can change priorities without mutating authored NPC identity. Obligations can pull NPCs toward work, social need can favour contact, disliked weather can discourage travel, avoided places are penalized, and closed businesses are strongly disfavoured.

Part 10 will build knowledge and gossip propagation on top of the encounters generated by these more purposeful routines.
