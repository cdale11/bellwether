Cumulative post-v0.1.0 runner note:
The runner's child stages reported Parts 1-20 passing in an earlier execution, but the aggregate process exceeded the execution window. This report does not misrepresent the aggregate runner as a clean timed completion. Focused release diagnostics and package-level checks were run independently.

Known performance boundary:
Free-form player-to-NPC dialogue remains synchronous by design because the player needs the immediate reply. Ollama itself is non-preemptive: a foreground request cannot interrupt a model generation already in progress. Bellwether therefore keeps background work serialized and bounded.
