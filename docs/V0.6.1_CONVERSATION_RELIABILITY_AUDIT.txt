Bellwether v0.6.1 Conversation Reliability Update — Audit Report

Scope
- Investigated user-supplied Ollama trace showing slow free dialogue, repeated NPC replies, overlong contextual narration, malformed format acceptance, and excessive social deltas.

Findings addressed
1. Free-dialogue output budget was too high for the desired conversational style and low-end CPU target.
2. Full prior NPC prose was injected more than once, increasing prompt size and repetition risk.
3. Small-model replies could repeat a prior answer despite prompt instructions.
4. Greetings and weather small talk could receive excessive trust/familiarity changes.
5. Dialogue parser recovery was not clearly distinguished in trace status from clean output.
6. Stored/retrieved recent dialogue could be character-truncated and re-injected as malformed prose.

Changes
- 48-token free-dialogue generation budget.
- One short sentence requested; 24-word engine ceiling.
- Compact recent conversation summaries replace duplicated verbatim reply injection.
- Similarity-based repetition detector with one corrective retry.
- Engine-side social clamps for greetings and short weather small talk.
- Repaired responses marked separately in trace metadata.
- Sentence/word-safe compact recent reply summaries.

Verification
- v0.6.1 conversation reliability diagnostic: 9/9 PASS.
- v0.6.1 travel/journey diagnostic: 11/11 PASS.
- v0.5.2 relationship diagnostic: 13/13 PASS.
- v0.5.0 core cast/memory diagnostic: 12/12 PASS.

- Python compilation: PASS.
- Frontend JavaScript syntax: PASS.

Limitations
- Live Ollama inference was not available in the build environment, so the exact i3-4160 latency improvement must be measured on the target machine.
- The compact continuity summary is deterministic and deliberately simple; later memory architecture can replace it with richer bounded semantic summaries without reintroducing full transcript injection.
- The 2B model may still occasionally produce weak prose; the validator now handles repetition and length, while future model routing can assign free dialogue to a stronger model where hardware permits.
