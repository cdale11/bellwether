Bellwether v1.0.3 focused audit

Scope: UI interaction hierarchy and local-LLM timeout efficiency.

Policy rationale:
- v1.0.1 traces showed strategic 4B requests timing out at 45 s and immediately retrying.
- Client timeout does not guarantee cancellation of Ollama-side generation; immediate retries can waste CPU.
- v1.0.2 compacted strategic prompts. v1.0.3 complements that work with one 120 s strategic attempt.
- Fast bounded choices use one 75 s attempt. Existing deterministic fallback and validation paths remain authoritative.

UI changes:
- Immediate story/danger/conversation actions remain directly visible.
- Routine actions remain grouped in contextual trays.
- Generic repetitive tick prose is omitted from the Recent Events rail, without changing authoritative history.
