BELLWETHER v0.6.1 — EXHAUSTIVE AUDIT REPORT

SCOPE
Travel and journey depth, save migration, route-state persistence, weather-sensitive travel timing, journey encounters, and Ollama CPU-thread utilization.

IMPLEMENTED
- Persistent travel state with route journey counts, familiarity, condition, last-travelled day and first-observation state.
- Weather-sensitive and condition-sensitive journey duration.
- Modest familiarity-based travel efficiency with a bounded maximum discount.
- First-time route observations for important authored routes and safe generic fallback text.
- Occasional bounded journey encounters based only on authoritative NPC location, weather and ecology context.
- Persistent bounded journey log and encounter history.
- Safe old-save migration.
- Ollama num_thread defaults to every CPU thread available to the process. Linux sched_getaffinity is preferred so CPU pinning/cgroup constraints are respected; os.cpu_count is the portable fallback. Developer override remains available.

BUG/CONSISTENCY AUDIT
- Confirmed movement still validates target against authoritative exits.
- Confirmed map discovery and path discovery remain applied after journey resolution.
- Confirmed first-observation state prevents repeated first-journey prose.
- Confirmed route familiarity cannot reduce journey time below the bounded design floor.
- Confirmed bad weather slows exposed routes.
- Confirmed travel state is JSON-serializable and migration-safe.
- Confirmed strategic/foreground model routing remains intact after CPU-thread change.
