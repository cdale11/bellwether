BELLWETHER v1.2.0 LIFE SIMULATION EXPANSION AUDIT

Audit scope
- Unpacked and inspected the complete v1.1.0 release tree before modification.
- Enumerated every packaged file and reviewed the source/document/content layout.
- Read every text source file as UTF-8 and scanned 29,477 text lines without decode errors after the v1.2.0 pass.
- Parsed every Python source file with compileall/AST-compatible compilation.
- Parsed every JSON catalogue and manifest.
- Syntax-checked the frontend JavaScript with Node.
- Scanned runtime sources for stale version strings, TODO/FIXME markers, exact duplicate files, obsolete runtime filenames and generated cache artefacts.
- Reviewed integration points in Game initialization, migration, public view, action generation, action execution, AI provider, AI player, full diagnostic, economy, jobs, activities, content progression, frontend panels and developer diagnostics.

Findings corrected
1. The diagnostic AI player could not certify ordinary NPC interaction because safe autonomous actions excluded story talk and free-talk used a separate UI path. Added bounded ordinary social contact actions that change relationship state without forcing authored story.
2. Goal-stall accounting treated any world-state change as progress, allowing unrelated actions to erase stagnation. Stagnation is now based on semantic goal progress.
3. v1.1.0 compact numeric action selection reused Ollama context across changing candidate maps. This could contaminate later numeric selections and is consistent with clustered invalid outputs. Volatile action-map calls no longer reuse persistent context.
4. Compact-choice parser failures lacked sufficient evidence. Parser stage, raw response excerpt, rejected index and candidate count are now traced.
5. The 45-second autonomous-player timeout was unnecessarily aggressive for transient contention. The default is now 90 seconds and remains environment-configurable; timeout failures reset disposable session state.
6. The diagnostic export filename still identified v1.1.0. Corrected to v1.2.0.
7. Full diagnostic direct recovery could inherit an incomplete old decision-accounting dictionary. Missing counters are now initialized defensively.
8. Life-simulation milestone needed stronger cross-system loops rather than isolated buttons. Added pantry preservation, shared meals, community participation, standing, long-term progression and hobby mastery while preserving deterministic authority.

Historical diagnostic note
Some old milestone-specific diagnostic scripts intentionally assert exact historical version numbers or superseded UI implementation strings. They are retained as historical evidence and may fail those exact-version/string assertions on a current release even when the preserved functionality is covered by current regression diagnostics. They were not used as current-release acceptance gates.

Release principle
v1.2.0 advances player-facing life simulation and automated certification together. The AI tester remains core development architecture, while deterministic tests retain authority over invariants and LLM agents test exploration, strategy, discoverability, behavioural quality and emergent progression.
