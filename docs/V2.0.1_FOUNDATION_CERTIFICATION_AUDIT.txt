BELLWETHER v2.0.1 — FOUNDATION CERTIFICATION AND PUBLIC-PATH CORRECTIONS

Method
------
The supplied v2.0.0 archive was treated as unverified. Inspection preceded editing. Work followed: inspect -> narrow hypothesis -> smallest surgical change -> syntax/contracts -> focused regression -> player-visible state path -> anomaly investigation -> evidence ledger -> packaging.

Observed before editing
-----------------------
1. The v2.0 architecture and VERSION=2.0.0 were present.
2. Python compilation, JavaScript syntax and JSON parsing passed.
3. Later subsystem diagnostics for living-world runtime, AI runtime architecture, temporal pacing, economy, life simulation, society and adaptive horror passed when invoked with the repository root on PYTHONPATH where required.
4. Several historical diagnostics failed because they assert their historical release version or superseded UI structure. Those failures are retained as evidence and were not hidden by rewriting old tests.
5. Bread consumption logic existed in PlayerStatusModel.perform but no bread action was emitted by PlayerStatusModel.actions, making the feature inaccessible through normal player action discovery.
6. Cottage repair execution accepted repair stages without enforcing location and stage prerequisites inside the model, allowing direct stale/fabricated calls to bypass the intended state machine.
7. /api/action passed arbitrary action IDs directly to Game.perform without checking the current legal public action surface.
8. AI overnight report text and report download filename contained hard-coded v2.0.0 metadata.

Surgical corrections
--------------------
- Expose status:eat:bread at Ashcroft Cottage when household bread_loaf > 0.
- Enforce cottage location and inspect -> prepare -> work prerequisites in PlayerStatusModel.perform.
- Guard /api/action against action IDs not present in Game.actions(); rejection returns the current view and does not execute the rejected gameplay action.
- Derive AI soak-report version text and download filename from VERSION.
- Add tools/v201_foundation_certification_diagnostic.py.

Focused executable evidence
---------------------------
The v2.0.1 diagnostic passed 14/14 checks:
- bread action player-visible;
- bread consumption changes authoritative hunger;
- repair inspect exposed;
- repair prepare follows inspect;
- repair work follows preparation;
- repair improves persistent cottage condition;
- remote repair rejected;
- fishing action player-visible;
- successful fishing writes inventory;
- quest reward transaction exactly once;
- portable save round-trip;
- release version 2.0.1;
- public API stale/illegal action guard present;
- AI report version dynamic.

Evidence limits
---------------
This release does not claim that every historical diagnostic passes unchanged. Some historical scripts encode exact old-version expectations or superseded UI selectors. They remain in the package for lineage evidence. This release also does not claim a real multi-hour Ollama overnight soak was completed in this build environment. AI architecture and stop semantics are preserved from v2.0.0; the focused v2.0.1 release verifies the corrected public/state contracts listed above.

Release decision
----------------
v2.0.1 is a corrective certification release, not a new gameplay expansion. The next major feature milestone should only begin after user runtime/playtest evidence from this package is reviewed, especially autonomous long-run behaviour and player-visible economy/life-loop balance.
