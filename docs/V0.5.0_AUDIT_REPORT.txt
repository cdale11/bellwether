AUDIT FIXES MADE
- Historical diagnostics that incorrectly treated the original three-person cast as permanently exact were generalized to validate an expanded complete core cast while preserving their original invariants.
- Historical version-lineage diagnostics were extended to accept the v0.5.x branch.
- A latent retry-path bug in AIProvider.ask_value referenced a trace variable that did not exist in that method. It was corrected so transport failures use provider status for retry cooldown selection.
- run.sh now reports both foreground and strategic model configuration and warns separately when the optional strategic model is missing.

KNOWN NON-BLOCKING LIMITATIONS
- Asha, Tom and Ruth currently use the generic bounded NPC action library. Their authored identity, needs, obligations, location constraints and purpose ranking already constrain decisions, but bespoke action libraries can make their daily lives more distinctive in a later social-content pass.
- Structured durable memory currently records player/NPC conversation events directly. The substrate supports more event types, but broader ingestion should be added deliberately as systems gain causal hooks rather than recording every simulation tick.
- The deep model route is configured but Town Mind/procedural strategic task classes are not yet scheduled. This is intentional roadmap discipline, not a missing v0.5.0 feature.
- Live Qwen QA was not used for final certification in this build environment. Deterministic fallback, parser isolation and routing configuration were tested. The README documents target-machine model installation and the release-candidate command for live-model QA.
