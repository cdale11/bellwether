# Bellwether v2.4.0 Audit — NPC Lives & Autonomous Change

## Pre-edit findings
The core cast already had authored identities, needs, obligations, purpose scoring, schedules, persistent social edges, encounter history, bounded knowledge propagation, memory and cognition. Lightweight residents already had persistent identities, households, occupations, schedules, co-location social links, and society-level employment/migration summaries. The missing gap was a persistent autonomous life-progression layer: personal goals were authored but not advancing, and lightweight residents rarely changed circumstances.

## Implementation boundary
Added `backend/core/npc_life_model.py` as a deterministic authoritative life-change layer. It progresses bounded core-cast personal goals from existing needs, records sparse resident employment/household-pressure events, exposes player influence hooks, and persists an event ledger. It does not mutate story canon, authored identity, romance canon, marriage, births, deaths, or move residents arbitrarily. Those require later authored/systemic work.

## Cleanup
Removed generated bytecode/cache and stale live diagnostic output before packaging. Historical diagnostics and prior audits retained as provenance/regression assets.
